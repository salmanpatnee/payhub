**show() implementation — CRITICAL security rules:**
```php
public function show(Payment $payment): Response|RedirectResponse
{
    $payment->loadMissing(['brand', 'stripeAccount']);

    // D-03 + D-12: Guard check BEFORE any StripeClient call
    if ($payment->status !== 'pending') {
        return Inertia::render('ClientPayment/Unavailable', [
            'status' => $payment->status,
            'brand'  => $this->brandProps($payment->brand),
        ]);
    }

    // D-01: Per-account StripeClient — NEVER Stripe::setApiKey() globally
    // secret_key auto-decrypted by Laravel encrypted cast
    $stripe = new StripeClient($payment->stripeAccount->secret_key);
    $pi = $stripe->paymentIntents->create([
        'amount'                    => $payment->amount,   // integer cents from DB — SEC-02
        'currency'                  => $payment->currency, // 'usd' or 'gbp'
        'automatic_payment_methods' => ['enabled' => true], // handles 3DS automatically — CLIENT-05
    ]);

    // D-02: Store PI ID so Phase 6 webhook handler can look up the Payment
    $payment->update(['stripe_payment_intent_id' => $pi->id]);

    // SEC-04: clientSecret only in Inertia props — NEVER logged, NEVER in URL
    return Inertia::render('ClientPayment/Pay', [
        'payment'       => $this->paymentProps($payment),
        'brand'         => $this->brandProps($payment->brand),
        'stripeAccount' => ['publishable_key' => $payment->stripeAccount->publishable_key],
        'clientSecret'  => $pi->client_secret,
    ]);
}
```

**success() implementation:**
```php
public function success(Payment $payment): Response|RedirectResponse
{
    // D-04: Stripe redirects with ?redirect_status=succeeded on success
    // SEC-04: Only read redirect_status — discard payment_intent_client_secret query param entirely
    if (request('redirect_status') !== 'succeeded') {
        return redirect()->route('pay.failed', $payment->uuid);
    }

    $payment->loadMissing('brand');

    return Inertia::render('ClientPayment/Success', [
        'payment' => [
            'amount'   => $payment->amount,
            'currency' => $payment->currency,
            'service'  => $payment->service,
        ],
        'brand' => $this->brandProps($payment->brand),
    ]);
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public internet → Laravel router | Any unauthenticated user can reach /pay/{uuid} — no session required |
| Laravel controller → Stripe API | Controller calls Stripe with decrypted secret_key |
| Stripe API response → Inertia props | client_secret travels from Stripe response to browser props |
| Stripe redirect → success controller | Stripe appends payment_intent_client_secret to return_url query string |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | Amount in PaymentIntent create() | mitigate | Amount read from `$payment->amount` (DB integer) only — no amount accepted from request body (CLAUDE.md rule enforced in show()) |
| T-05-02 | Information Disclosure | client_secret exposure | mitigate | SEC-04: clientSecret only in Inertia props for ClientPayment/Pay render — not logged, not in URL, not in session. success() reads only `redirect_status`, discards `payment_intent_client_secret` query param. |
| T-05-03 | Spoofing | Direct navigation to /pay/{uuid}/success | mitigate | success() verifies `request('redirect_status') === 'succeeded'` before rendering; redirects to failed if absent or wrong value |
| T-05-04 | Tampering | Cross-account key contamination | mitigate | Per-request `new StripeClient($account->secret_key)` — NEVER global Stripe::setApiKey(). Enforced by CLAUDE.md and verified by grep gate. |
| T-05-05 | Information Disclosure | UUID enumeration | accept | UUID v4 has 2^122 possible values — not practically enumerable. No rate-limiting needed for v1. |
| T-05-06 | Tampering | Non-pending payment re-processing | mitigate | D-03/D-12: Guard check runs before StripeClient instantiation — completed/failed/cancelled payments never get new PaymentIntents |
</threat_model>
