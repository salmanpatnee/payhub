<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem → Git | Source code, package manifests, .env.example committed to repo. .env (with real APP_KEY) never committed. |
| Operator → APP_KEY | The APP_KEY env var is the single secret that decrypts encrypted casts. Loss of APP_KEY = permanent loss of all encrypted credentials. |
| npm/Packagist → local node_modules/vendor | Third-party packages introduce supply-chain risk. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-01 | Information Disclosure | `.env` accidentally committed exposing APP_KEY | mitigate | Default Laravel `.gitignore` excludes `.env`; this task ONLY edits `.env.example` (template, no secrets). Acceptance criterion grep verifies `.env` does NOT contain APP_PREVIOUS_KEYS placeholder strings, ensuring the two files remain distinct. |
| T-01-01-02 | Denial of Service / Loss of Data | Operator rotates APP_KEY without setting APP_PREVIOUS_KEYS first → all encrypted Stripe credentials become irrecoverable | mitigate | D-12 placeholder in `.env.example` with explicit warning ("permanently lose all encrypted Stripe credentials") visible to anyone copying the template. Acceptance criterion grep enforces presence of the warning text. |
| T-01-01-03 | Tampering | Malicious package version pinned during scaffold (supply chain) | mitigate | Pin exact major versions per RESEARCH.md verified-on-Packagist constraints (`^1.37`, `^7.4`, etc.). composer.lock and package-lock.json committed to git pin transitive deps. Use only `laravel/*`, `spatie/*`, `stripe/*`, `inertiajs/*`, official `shadcn-vue` registry — no third-party registries. |
| T-01-01-04 | Tampering | Wrong shadcn-vue style or third-party component registry introduces unreviewed code | mitigate | UI-SPEC.md "Registry Safety" section locks Phase 1 to the official shadcn-vue.com registry only. The five seed components (button, input, label, card, badge) come exclusively from the official registry. Pitfall 3 mitigation prevents accidental re-init that would change `components.json` style. |
| T-01-01-05 | Information Disclosure | Telescope dev tool installed in production accidentally exposes request/job internals | accept | Telescope is installed as a dev-only dependency (`composer require --dev`). Production deploys MUST use `composer install --no-dev`. This is a deferred concern (production hardening) outside Phase 1 scope; Phase 1 dev environment risk is acceptable. |

</threat_model>
