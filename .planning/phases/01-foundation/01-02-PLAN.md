```php
Schema::create('stripe_accounts', function (Blueprint $table) {
    $table->id();
    $table->foreignId('brand_id')->constrained()->cascadeOnDelete();
    $table->string('account_name');
    $table->string('publishable_key');   // plain text — public key, no encryption
    $table->text('secret_key');          // D-10: TEXT, encrypted cast — NEVER VARCHAR
    $table->text('webhook_secret');      // D-10: TEXT, encrypted cast — NEVER VARCHAR
    $table->boolean('is_active')->default(true);
    $table->timestamps();
});
```

**payments table (D-06, D-07, D-09 — integer cents, ENUM status, nullable timestamps):**
```php
Schema::create('payments', function (Blueprint $table) {
    $table->id();
    $table->uuid('uuid')->unique();
    $table->foreignId('brand_id')->constrained()->cascadeOnDelete();
    $table->foreignId('stripe_account_id')->constrained()->cascadeOnDelete();
    $table->foreignId('user_id')->constrained()->cascadeOnDelete();
    $table->unsignedBigInteger('amount');                   // integer cents — NEVER float or decimal
    $table->string('currency', 3);                          // 'usd' or 'gbp'
    $table->string('description')->nullable();
    $table->enum('status', ['pending', 'completed', 'failed', 'cancelled'])->default('pending'); // D-07
    $table->string('client_email');
    $table->string('stripe_payment_intent_id')->nullable();
    $table->timestamp('expires_at')->nullable();            // D-09: NULL = never expires
    $table->timestamp('paid_at')->nullable();
    $table->timestamps();
});
```

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Eloquent cast → MySQL TEXT column | The `encrypted` cast serializes plaintext to ciphertext before writing. The boundary is the model layer: anything reading the raw DB value (direct SQL, mysqldump, DB inspection tools) sees only ciphertext. |
| APP_KEY → encrypted cast | The APP_KEY is the sole secret that encrypts and decrypts credential columns. Its loss or unauthorized access is the primary risk surface for this plan. |
| Seeder → Database | DatabaseSeeder writes placeholder credentials to the database. These credentials are fake (sk_test_placeholder_for_dev_only) and must never be replaced with real keys via seeder code committed to git. |
| Test database → Production database | Pest tests with RefreshDatabase use the test database (configured in phpunit.xml). Tests must never touch the production database. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | Tampering | `secret_key` column typed as VARCHAR instead of TEXT in stripe_accounts migration | mitigate | `acceptance_criteria` for Task 1 includes `grep -n "table->text('secret_key')"` that must match — VARCHAR column silently truncates encrypted ciphertext causing DecryptException or partial decryption returning wrong plaintext. Plan explicitly forbids `$table->string()` for these columns. |
| T-01-02-02 | Tampering | `amount` column typed as `decimal` or `float` instead of `unsignedBigInteger` | mitigate | `acceptance_criteria` for Task 1 requires `grep -n "table->unsignedBigInteger('amount')"` to match. PaymentAmountIntegrityTest uses `toBeInt()` assertion to catch float drift at the model layer. |
| T-01-02-03 | Information Disclosure | Seeder committed to git with real Stripe secret keys in place of placeholders | mitigate | Task 3 action specifies literal placeholder strings (`sk_test_placeholder_for_dev_only`, `whsec_placeholder_for_dev_only`). Acceptance criterion grep checks these exact strings exist in StripeAccountFactory and seeder. CI must never override with real keys. |
| T-01-02-04 | Information Disclosure | Database dump exposes Stripe credentials without APP_KEY | accept | Accepted by design — the Laravel `encrypted` cast (AES-256-CBC + MAC via APP_KEY) is the control. | T-01-02-05 | Elevation of Privilege | Seeder assigns `admin` role before `admin` Role record exists in the roles table | mitigate | DatabaseSeeder seeds `Role::firstOrCreate(['name' => 'admin'])` and `Role::firstOrCreate(['name' => 'user'])` BEFORE any `syncRoles()` call. The `stripe_payment_intent_id` column stores only the non-secret `pi_xxx` reference. |
