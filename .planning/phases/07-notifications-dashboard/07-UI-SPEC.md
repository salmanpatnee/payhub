## Visual Hierarchy

Primary visual anchor: the payments table card. Secondary anchor: the "New payment" primary CTA button in the page header.

The filter bar sits above the table card and is visually subordinate — it uses no accent color and no elevated surface. Its role is utilitarian: to narrow the primary anchor's data set.

The existing `<Eye>` and `<Copy>` icon-only row action buttons are inherited from `payments/Index.vue` with their existing accessibility labels — they are not new introductions in this phase and are not reassigned accent color.

## Interaction Contract

### Filter Auto-Submit Behavior

- Every filter change triggers `router.get(route('payments.index'), activeFilters, { preserveState: true, replace: true })` immediately.
- Empty string values are stripped before submission: `Object.fromEntries(Object.entries(filters).filter(([, v]) => v !== ''))`.
- `replace: true` prevents history accumulation per keystroke.
- `preserveState: true` maintains scroll position.
- No debounce needed — `<Select>` fires only on selection commit; `<input type="date">` fires on blur/change (not keypress).

### Clear All Filters

- A small `<Button variant="ghost" size="sm">` with `<X class="size-4" />` and visible text "Clear filters" appears at the end of the filter row when any filter has a value.
- The button carries `aria-label="Clear all filters"` for screen readers.
- Clicking sets all filter values to `''` and triggers the watcher (which then submits with empty params, navigating to `/payments`).
- Conditional render: `v-if="hasActiveFilters"` where `hasActiveFilters = computed(() => Object.values(filters).some(v => v !== ''))`.

### Filter State Persistence

- Controller passes `filters` prop back via `Inertia::render()`.
- Vue initialises reactive state as `reactive({ ...props.filters })`.
- Filter inputs are pre-populated on every page load and back-navigation.

## Accessible Interaction Notes

- Filter `<Select>` components must render with an associated `<Label>` for screen reader association — use `<Label :for="'brand-filter'">Brand</Label>` paired with a native `id` prop if needed, or rely on shadcn-vue's built-in `aria-label` on `<SelectTrigger>`.
- Date inputs: use native `<input type="date">` wrapped in shadcn `<Input>` — native date pickers are accessible by default.
- Clear filters button: include `aria-label="Clear all filters"` on the ghost button.
- Status badge `<Badge variant="outline">` contains only the status text — sufficient for screen readers.
- `<Eye>` and `<Copy>` icon-only row action buttons retain their existing `aria-label` values from `payments/Index.vue` — do not change them.
