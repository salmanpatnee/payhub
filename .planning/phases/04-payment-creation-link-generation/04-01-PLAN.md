**Step 1: Create `app/Http/Requests/StorePaymentRequest.php`.**

```php
<?php

namespace App\Http\Requests;

use Illuminate\Foundation\Http\FormRequest;
use Illuminate\Validation\Rule;

class StorePaymentRequest extends FormRequest
{
    public function authorize(): bool
    {
        // Both Admin and User can create payments (D-03)
        return $this->user() !== null;
    }

    public function rules(): array
    {
        return [
            'brand_id'          => ['required', 'integer', 'exists:brands,id'],
            'stripe_account_id' => ['required', 'integer',
                                    Rule::exists('stripe_accounts', 'id')
                                        ->where('is_active', true)],
            'currency'          => ['required', 'string', 'in:usd,gbp'],
            'amount'            => ['required', 'numeric', 'min:0.01'],
            'client_name'       => ['required', 'string', 'max:255'],
            'client_email'      => ['required', 'email', 'max:255'],
            'service'           => ['nullable', 'string', 'max:255'],
            'package'           => ['nullable', 'string',
                                    'in:basic,standard,premium,platinum,diamond'],
            'note'              => ['nullable', 'string'],
        ];
    }

    protected function passedValidation(): void
    {
        // SEC-02: Decimal input converted to integer cents server-side.
        // This is the ONLY place the client-submitted amount is transformed.
        // The controller reads $request->validated()['amount'] which is already integer cents.
        $this->merge(['amount' => (int) round($this->amount * 100)]);
    }
}
```

**Key constraints:**
- `user_id` and `status` are NOT in rules() — they are set explicitly in the controller, never from client input.
- `passedValidation()` is the only place amount conversion happens. The controller must use `$request->validated()`, never `$request->amount` directly.
- `Rule::exists('stripe_accounts', 'id')->where('is_active', true)` — the `where()` clause uses boolean `true`. If SQLite stores booleans as 0/1 integers and `where('is_active', true)` fails, use `->where('is_active', 1)` as the fallback (see RESEARCH.md Assumption A2).

**Step 2: Create `app/Http/Controllers/PaymentController.php`.**

```php
<?php

namespace App\Http\Controllers;

use App\Http\Requests\StorePaymentRequest;
use App\Models\Brand;
use App\Models\Payment;
use App\Models\StripeAccount;
use Illuminate\Http\RedirectResponse;
use Inertia\Inertia;
use Inertia\Response;

class PaymentController extends Controller
{
    public function index(): Response
    {
        $user  = auth()->user();
        $query = Payment::with(['brand', 'stripeAccount', 'user'])
                        ->orderByDesc('created_at');

        if (! $user->hasRole('admin')) {
            $query->where('user_id', $user->id);
        }

        return Inertia::render('payments/Index', [
            'payments' => $query->get()->map(fn (Payment $p) => [
                'id'           => $p->id,
                'uuid'         => $p->uuid,
                'amount'       => $p->amount,       // integer cents — formatted in Vue
                'currency'     => $p->currency,
                'brand_name'   => $p->brand->name,
                'account_name' => $p->stripeAccount->account_name,
                'status'       => $p->status,
                'created_at'   => $p->created_at->toISOString(),
                'client_email' => $p->client_email,
                'client_name'  => $p->client_name,
            ]),
        ]);
    }

    public function create(): Response
    {
        return Inertia::render('payments/Create', [
            'brands'         => Brand::orderBy('name')->get(['id', 'name']),
            'stripeAccounts' => StripeAccount::where('is_active', true)
                                             ->orderBy('account_name')
                                             ->get(['id', 'account_name']),
        ]);
    }

    public function store(StorePaymentRequest $request): RedirectResponse
    {
        // $request->validated() contains amount already converted to integer cents
        // by StorePaymentRequest::passedValidation() — SEC-02 guarantee.
        // user_id and status are NEVER from the request.
        $payment = Payment::create([
            ...$request->validated(),
            'user_id'    => auth()->id(),
            'status'     => 'pending',
            'expires_at' => null,          // PAY-07: links never expire
        ]);

        return redirect()->route('payments.show', $payment);
        // Route model binding resolves via uuid (getRouteKeyName = 'uuid')
    }

    public function show(Payment $payment): Response
    {
        return Inertia::render('payments/Show', [
            'payment' => [
                'uuid'         => $payment->uuid,
                'amount'       => $payment->amount,   // integer cents — formatted in Vue
                'currency'     => $payment->currency,
                'status'       => $payment->status,
                'client_name'  => $payment->client_name,
                'client_email' => $payment->client_email,
                'service'      => $payment->service,
                'package'      => $payment->package,
                'note'         => $payment->note,
                'brand_name'   => $payment->brand->name,
                'created_at'   => $payment->created_at->toISOString(),
            ],
        ]);
    }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client browser → POST /payments | Untrusted form data including amount, currency, brand_id, stripe_account_id |
| FormRequest → Controller | Only $request->validated() crosses this boundary — raw $request never used in controller |
| Auth middleware → Controller | Unauthenticated requests redirect to login before reaching controller |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-01 | Tampering | StorePaymentRequest | mitigate | Amount conversion in passedValidation() ONLY — controller reads $request->validated()['amount'] (integer cents). No $request->amount in controller. Enforced by spreading ...$request->validated() into Payment::create(). |
| T-4-02 | Tampering | StorePaymentRequest stripe_account_id | mitigate | Rule::exists('stripe_accounts', 'id')->where('is_active', true) ensures deactivated accounts cannot be submitted even if ID is guessed. Returns 422. |
| T-4-03 | Tampering | PaymentController::store() mass assignment | mitigate | user_id and status set explicitly after spread of validated() — user_id = auth()->id() not from request. status = 'pending' hardcoded. Neither field appears in StorePaymentRequest::rules(). |
| T-4-04 | Tampering | StorePaymentRequest currency validation | mitigate | 'in:usd,gbp' rule rejects any other currency string. Returns 422 on invalid currency. |
| T-4-05 | Information Disclosure | PaymentController::show() | accept | Show page at /payments/{uuid} is under auth+verified middleware — unauthenticated access redirects to login. Client-facing page /pay/{uuid} is a separate Phase 5 concern. |
</threat_model>
