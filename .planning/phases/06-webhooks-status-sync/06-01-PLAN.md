Step 2: Replace the generated stub with this exact implementation in `app/Http/Controllers/StripeWebhookController.php`:

```php
<?php

namespace App\Http\Controllers;

use App\Jobs\HandleStripeWebhookJob;
use App\Models\Payment;
use App\Models\StripeAccount;
use Illuminate\Http\Request;
use Illuminate\Http\Response;
use Stripe\Exception\SignatureVerificationException;
use Stripe\Webhook;

class StripeWebhookController extends Controller
{
    private const HANDLED_EVENTS = [
        'payment_intent.succeeded',
        'payment_intent.payment_failed',
    ];

    public function handle(Request $request, StripeAccount $stripeAccount): Response
    {
        $payload   = $request->getContent();
        $sigHeader = $request->header('Stripe-Signature', '');

        try {
            $event = Webhook::constructEvent($payload, $sigHeader, $stripeAccount->webhook_secret);
        } catch (SignatureVerificationException|\UnexpectedValueException) {
            return response('Invalid signature or payload', 400);
        }

        if (! in_array($event->type, self::HANDLED_EVENTS)) {
            return response('', 200);
        }

        $piId    = $event->data->object->id ?? null;
        $payment = $piId ? Payment::where('stripe_payment_intent_id', $piId)->first() : null;

        // Idempotency gate 1: skip if already in terminal state
        if ($payment && in_array($payment->status, ['completed', 'failed'])) {
            return response('', 200);
        }

        HandleStripeWebhookJob::dispatch(
            $stripeAccount->id,
            $event->type,
            $event->data->object->toArray(),
        );

        return response('', 200);
    }
}
```

Key rules:
- `$request->getContent()` — raw body string. NEVER `$request->all()` or `$request->json()`.
- `$stripeAccount->webhook_secret` — auto-decrypted by Eloquent 'encrypted' cast. NEVER `getRawOriginal()`.
- Both `SignatureVerificationException` AND `\UnexpectedValueException` caught — both return 400.
- `$event->data->object->toArray()` — serialize PI data for the job.
- No auth middleware applied — this is a public controller.
- The controller does NOT call `Stripe::setApiKey()` — `Webhook::constructEvent()` is a static utility, no API key needed.

| Internet → webhook route | Stripe POSTs raw JSON; any attacker can also POST |
| webhook route → database | Job writes payment status; controller must verify before dispatching |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing / Tampering | StripeWebhookController::handle() | mitigate | `Webhook::constructEvent()` HMAC-SHA256 check; tampered or missing sig → 400 |
| T-06-02 | Tampering | StripeWebhookController::handle() raw body | mitigate | `$request->getContent()` (raw bytes); NEVER `$request->all()` — re-encoding breaks HMAC |
| T-06-04 | Information Disclosure | StripeAccount::webhook_secret in controller | mitigate | Access via Eloquent model property only (auto-decrypted by 'encrypted' cast); getRawOriginal() forbidden |
| T-06-05 | Availability | Webhook route placement | mitigate | Route added outside ALL middleware groups in routes/web.php; Stripe servers have no auth session |
| T-06-06 | Availability | CSRF middleware on webhook route | mitigate | `preventRequestForgery(except: ['webhook/stripe/*'])` in bootstrap/app.php; verified L13 API |
