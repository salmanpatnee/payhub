## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap in nav items, inline badge padding |
| sm | 8px | Form field internal padding top/bottom, row gaps within a grid |
| md | 16px | Default form field spacing, card padding, sidebar group padding |
| lg | 24px | Section spacing between form groups, page content padding |
| xl | 32px | Major vertical spacing between page sections |
| 2xl | 48px | Auth page vertical centering gap |
| 3xl | 64px | Not used in Phase 2 |

Exceptions:
- Touch targets on interactive sidebar items: minimum 44px height (achieved via shadcn SidebarMenuButton `size="lg"` — already in scaffold)
- Auth page max-width container: 384px (`max-w-sm`) — matches existing AuthSimpleLayout.vue

## Typography

All type is set in Instrument Sans as declared in `app.css`. The CSS variable `--font-sans` resolves to `Instrument Sans, ui-sans-serif, system-ui, sans-serif`.

| Role | Size | Weight | Line Height | Tailwind Class | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` | Table cell content, form helper text, descriptive copy |
| Label | 14px | 500 (medium) | 1.4 | `text-sm font-medium` | Form field labels, table column headers |
| Heading | 20px | 500 (medium) | 1.2 | `text-xl font-medium` | Page titles, auth card heading (matches existing AuthSimpleLayout.vue `text-xl font-medium`) |
| Display | 28px | 600 (semibold) | 1.2 | `text-2xl font-semibold` | Not used in Phase 2 — reserved for Phase 7 dashboard |

Active weights in this phase: **400 (regular)** and **500 (medium)**. Weight 600 is not used until Phase 7.

Muted text for secondary content: `text-muted-foreground` (`hsl(0 0% 45.1%)` in light mode) — used for descriptions, placeholder text, helper copy.

## Color

All colors resolved from CSS custom properties defined in `app.css`. Values below are the light-mode defaults.

| Role | CSS Token | Light Mode Value | Usage |
|------|-----------|-----------------|-------|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` — white | Page background, main content area |
| Secondary (30%) | `--sidebar-background` | `hsl(0 0% 98%)` — off-white | Sidebar surface; `--card` (`hsl(0 0% 100%)`) for form cards |
| Accent (10%) | `--primary` | `hsl(0 0% 9%)` — near-black | Primary CTA button fill, active sidebar nav item indicator |
| Destructive | `--destructive` | `hsl(0 84.2% 60.2%)` — red | Delete user button, delete confirmation dialog |

Accent (`--primary`) reserved for:
1. "Log in" submit button (Button default variant)
2. "Create user" submit button on Create form
3. "Save changes" submit button on Edit form
4. Active/selected sidebar navigation item highlight

Secondary accent (`--muted-foreground`) for disabled states and placeholder text only.

Badge colors for role display:
- Admin role badge: `variant="secondary"` (uses `--secondary` surface with `--secondary-foreground` text) — neutral, not alarming
- User role badge: `variant="outline"` — distinguished from admin without adding color noise

Destructive state reserved exclusively for:
- Delete user button in the users table row actions
- Delete confirmation dialog confirm button

Status message colors (flash messages via Sonner):
- Success: Sonner default success variant (green indicator)
- Error: Sonner default error variant (red indicator)

## Page-Level Specifications

### Login Page (`auth/Login.vue`) — No Changes Required

The existing Login.vue already implements the correct design. Confirm:
- `canRegister` prop: when Fortify registration is disabled, `canRegister` becomes `false` and the "Don't have an account? Sign up" link disappears automatically — no template change
- Status message: green `text-green-600 text-sm font-medium text-center` — keep as-is
- Layout: `AuthSimpleLayout.vue` with centered card, max-w-sm, logo at top

### Dashboard (`pages/Dashboard.vue`) — No Changes Required

The existing Dashboard.vue with `PlaceholderPattern` tiles is the correct stub for Phase 2. It uses `AppSidebarLayout` via `defineOptions({ layout: { breadcrumbs: [...] } })`.

### User List (`admin/users/Index.vue`)

Layout wrapper: AppSidebarLayout with breadcrumbs: `[{ title: 'Users', href: '/admin/users' }]`

**Focal point:** Primary focal point is the "Add user" CTA button in the top-right of the page header, as the primary action Admin performs on this screen.

Page structure:
```
<div class="p-6 space-y-6">
  <!-- Page header -->
  <div class="flex items-center justify-between">
    <h1 class="text-xl font-medium">Team Members</h1>
    <Button as-child><Link href="/admin/users/create">Add user</Link></Button>
  </div>

  <!-- User table -->
  <table class="w-full text-sm">
    <thead>
      <tr class="border-b border-border">
        <th class="text-left py-3 pr-4 font-medium text-muted-foreground">Name</th>
        <th class="text-left py-3 pr-4 font-medium text-muted-foreground">Email</th>
        <th class="text-left py-3 pr-4 font-medium text-muted-foreground">Role</th>
        <th class="text-right py-3 font-medium text-muted-foreground">Actions</th>
      </tr>
    </thead>
    <tbody>
      <!-- Row per user -->
      <tr class="border-b border-border last:border-0">
        <td class="py-3 pr-4">{{ user.name }}</td>
        <td class="py-3 pr-4 text-muted-foreground">{{ user.email }}</td>
        <td class="py-3 pr-4">
          <Badge :variant="user.roles.includes('admin') ? 'secondary' : 'outline'">
            {{ user.roles[0] ?? 'user' }}
          </Badge>
        </td>
        <td class="py-3 text-right">
          <!-- Edit + Delete row actions as icon buttons or DropdownMenu -->
        </td>
      </tr>
    </tbody>
  </table>
</div>
```

Row actions: Two icon buttons (edit icon links to edit page, trash icon triggers delete confirmation). No DropdownMenu needed at this scale.
- Edit: `<Button variant="ghost" size="icon" as-child><Link :href="editRoute"><Pencil class="size-4" /></Link></Button>`
- Delete: `<Button variant="ghost" size="icon" @click="confirmDelete(user)"><Trash2 class="size-4 text-destructive" /></Button>`

Delete confirmation: Dialog component — "Delete user?" heading, "This will permanently remove {name}'s account." body, "Keep account" (outline) + "Delete user" (destructive variant) buttons.

Empty state: When `users` array is empty:
```
<tr>
  <td colspan="4" class="py-12 text-center text-muted-foreground text-sm">
    No team members yet. Add the first user above.
  </td>
</tr>
```

Pagination: Full list — no pagination. Team is small (internal tool). Server sends all users ordered by name.

### Create User (`admin/users/Create.vue`)

Layout wrapper: AppSidebarLayout with breadcrumbs: `[{ title: 'Users', href: '/admin/users' }, { title: 'Add user', href: '/admin/users/create' }]`

Form structure: Single card, no tabs.

```
<div class="p-6 max-w-lg">
  <Card>
    <CardHeader>
      <CardTitle>Add team member</CardTitle>
      <CardDescription>Create a new account. The user can change their password after logging in.</CardDescription>
    </CardHeader>
    <CardContent>
      <form class="space-y-4">
        <!-- Name field -->
        <!-- Email field -->
        <!-- Password field (plain Input, not PasswordInput toggle — admin sets, user changes later) -->
        <!-- Role select -->
      </form>
    </CardContent>
    <CardFooter class="flex justify-between">
      <Button variant="outline" as-child><Link href="/admin/users">Back to users</Link></Button>
      <Button type="submit" :disabled="processing">
        <Spinner v-if="processing" />
        Create user
      </Button>
    </CardFooter>
  </Card>
</div>
```

Role select options: "admin" (displayed as "Admin") and "user" (displayed as "User"). Default selection: "user".

Validation errors: `InputError` component below each field (same pattern as Login.vue).

### Edit User (`admin/users/Edit.vue`)

Layout wrapper: AppSidebarLayout with breadcrumbs: `[{ title: 'Users', href: '/admin/users' }, { title: user.name, href: '#' }]`

Form structure: Same card layout as Create. Password field is optional on edit — placeholder text "Leave blank to keep current password".

Role select: Pre-populated with user's current role.

Delete action: "Delete user" Button (`variant="destructive"`) in a separate danger zone section below the card, OR in the CardFooter left-aligned. Preference: in CardFooter left side, "Back to users" center-ish, Save right — standard three-action footer layout. Delete triggers Dialog confirmation before submitting DELETE request.

Self-delete guard: When `user.id === auth.user.id`, hide the delete button entirely from the UI (backend also guards this, but UI should not present the option).

### Coming Soon (`placeholders/ComingSoon.vue`)

Minimal full-height centered placeholder — no card wrapper.

```
<div class="flex h-full items-center justify-center">
  <div class="text-center space-y-2">
    <p class="text-muted-foreground text-sm">This section is coming in a future update.</p>
  </div>
</div>
```

No icon, no heading, no button. Purely informational — this is internal and ephemeral.

## Interaction States

### Form Submission

| State | Visual |
|-------|--------|
| Idle | Button enabled, normal label |
| Processing | Button disabled, Spinner prepended to label (matches Login.vue pattern using `<Spinner v-if="processing" />`) |
| Success | Redirect to Index with Sonner toast: success variant |
| Validation error | Stay on page; InputError shown below each invalid field; no toast |
| Server error (500) | Inertia default error page — no custom handling in Phase 2 |

### Navigation Active State

`isCurrentUrl()` composable (already in `NavMain.vue`) handles active highlighting via SidebarMenuButton `:is-active` prop. No additional implementation needed.

### Sidebar Collapsed State

Sidebar uses `collapsible="icon"` (already in AppSidebar.vue scaffold). In collapsed state:
- Nav items show icon only; Tooltip shows title on hover (handled by SidebarMenuButton `:tooltip` prop — already wired)
- Admin-only "Users" item disappears from collapsed view if `isAdmin` is false (same `v-if` logic applies)

## Accessibility Notes

- All form fields have explicit `<Label for="...">` with matching `id` on the input (existing Login.vue pattern — maintain for all admin forms)
- Delete icon buttons must have `aria-label="Delete {name}"` to provide accessible name
- Dialog component from shadcn-vue handles focus trap and `aria-modal` automatically
- Sidebar tooltip on collapsed state uses Tooltip component — keyboard accessible via existing SidebarMenuButton implementation
- Admin-only nav items hidden via `v-if` (not `v-show`) — hidden elements are fully removed from DOM and accessibility tree
